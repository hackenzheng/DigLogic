    实验原理

    1、 自顶向下的结构化设计方法

    自顶向下的结构化设计方法是一种软件工程中常用的设计方法，它也可以应用到数字电路设计中。在数字电路设计中，自顶向下的结构化设计方法指的是从系统级的功能需求开始，逐步细化为各个子模块或组件的设计，直至最终实现整个数字电路系统。这种方法有助于减少复杂系统的设计难度，提高设计效率。

    具体实现是指在设计过程中，设计开发人员首先从整体上规划整个系统的功能和性能，然后根据具体需求和系统功能对系统进行子模块划分，从而将整个系统分解为规模较小、功能较简单的子模块，并在这个过程中确立子模块之间的交互关系和接口。这个过程可以不断地进行下去，直到最终得到的子模块可以很容易地映射到物理层面实现。

    在数字电路设计中，首先将一个比较复杂的数字电路划分为多个组成模块，再分别对每个模块建模，然后将这些模块组成一个总模块，完成所需的功能。结构化设计的基本单位是模块，每个模块的组成可以遵循以下规则：

    （1）在设计中至少被使用两次的逻辑，该逻辑采用实例化方式重用，为增加代码的易读性和整洁性，可将其封装成模块，如译码器、多路选择器等；

    （2）功能规格十分明确，且与外界的交互信号数目不是很多的，封装成模块，如消抖模块等；

    （3）现有的一个模块达到了数千行代码的规模，将其拆分成若干和小模块；

    （4）一个文件中只包含一个模块，便于后期代码维护。


    下面以一个简单的计数器设计为例，说明自顶向下的结构化设计方法在数字电路实验中的应用原理：

    系统需求分析： 首先明确计数器的功能需求，比如设计一个能够实现从0到9循环计数的计数器。

    模块分解： 根据功能需求，将计数器系统分解为若干个子模块，比如计数模块、显示模块等。

    模块设计： 对每个子模块进行具体设计。例如，计数模块可以所给的计数器模板代码或者用状态机实现数字的递增；显示模块可以用数码管来显示计数结果。

    模块集成： 完成各个子模块的设计后，将它们集成到整个计数器系统中，确保各个模块之间的接口连接正确。

    系统测试与调试： 对整个系统进行测试，确保计数器在各种情况下都能正常工作。如果发现问题，需要回到相应的模块进行调试和修复。

    通过以上步骤，可以看到整个设计过程是自顶向下逐步细化的，从最初的系统级需求到最终的具体电路实现。这种方法可以帮助设计者在复杂的数字电路设计中有条不紊地进行设计，并保证最终的系统符合预期的功能要求。

    3、系统时钟分频

    Minisys开发板晶振的时钟输出为100MHz，但在开发的过程中，需要更高或更低频率的时钟，这时就需要时钟IP，将晶振输出的100MHz时钟，进行倍频或分频，从而得到准确且无毛刺的所需时钟。

    3.1 时钟IP建立

    在流程栏里，单击IP Catalog

    在右侧弹出的IP Catalog对话框里，搜索需要的IP，这里输入clock，然后选择Clocking Wizard项并双击。

    在弹出的Customize IP对话框里定制IP，首先给IP命名如clk_div，然后再在Clocking Options标签页，将时钟类型配置为PLL，其他保持默认即可

    在Output Clocks标签页，配置输出时钟，如20MHz，下方端口处需选择locked，用于指示PLL锁定时间，最后点击OK；

    在弹出来的生成IP界面，选择Generate；

    当IP综合完成后，在Design Sources中就出现了刚刚创建的时钟IP，clk_div；

    3.2 时钟IP例化

    创建好IP后，需要例化才能使用，时钟IP clk_div的例化如下：

    clk_div u_clk_div (
    .clk_in1  (clk   ),
    .clk_out1 (clk_g ),
    .locked   (locked)
);

    IP的例化名需要与IP名相同，且例化端口需要与IP提供的端口名相同。例化好后，在Design Sources里会发现，生成的IP已经被集成进去。

    时钟IP例化后，综合可能报以下错误， [DRC REQP-1712] Input clock driver: Unsupported PLLE2_ADV connectivity.

    可以尝试将输入设置为Global buffer
